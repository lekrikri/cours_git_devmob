Cours GIT

Qu'est-ce que GIT

GIT est un outil de versionning, et il permet d'avoir une tracabilite et un historique de tous les fichiers du projet

Il permet egalement de travailler en equipe: Envoyer nos modifications a nos collegues et recuperer les modifications des collegues>

Dans la tracabilite des fichiers GIT nous donne toujours 3 infos: Qui, Quand, Pourquoi?

Les commandes de base

Initialiser un projet:
git init
Voir le statut des fichiers
git status
Ajouter des fichiers
git add "fichier"
Ajouter TOUS LES FICHIERS :
git add .
Commit (enregistrer) les fichiers ajoutes
git commit
Un commit necessite toujours un message
commit avec message :
git commit -m "message"
commit vec add + message (fonctionne sur les fichiers deja add precedemment)
git commit -a -m "messaage"

Correction des fichiers

Voir les differents commit :
git log
Corriger les modifications sur un fichier (pas encore add) :
git checkout "fichier"
Supprimer un add :
git reset "fichier"
git reset
Supprimer le dernier commit :
git commit reset HEAD^^

La notion de branche

Une branche est une copie du projet. Elle permet de developper en paralelle de nouvelles fonctionalitees. Lorsqu'on considere les nouveautes operationnelles, on peut les rappatrier sur la version principale (master)

Voir les branche:
git branch
Creer une nouvelle branche :
git branch "nom"
Se placer sur une branche :
git checkout "nom"
Voir les differents entre deux branches :
git diff "nom"..master
Apporter les modif effectuees sur une branche dans master (il faut etre sur master) :
git checkout master
git merge "nom"
Remplacer master par une branche :
git checkout master
git fetch . "nom"
Le fetch va litteralement remplacer master par la branche. Il s'agit donc d'une grosse modification dans le projet (ex: version 2)
